Chinese (zh)
Uses Chinese numeral characters: 一二三四五六七八九十百千万亿… and in financial contexts the complex forms 壹贰叁肆伍陆柒捌玖拾佰仟萬億, etc.

Large numbers are structured in 10⁴ blocks: 10,000 = 万, 100,000,000 = 亿, so 12,345 is written 一万二千三百四十五, not grouped 12,345 as in Western formats.

Japanese (ja)
Traditionally uses kanji numerals: 一二三四五六七八九十百千万億兆….

Also groups conceptually by 10⁴: 10,000 = 万, 100,000,000 = 億, 1,0000,0000 (in 4‑digit chunks) = 一億 (“one hundred million”).

A purely traditional rendering of 1234567 would be 一百二十三万四千五百六十七, i.e. “123 ten‑thousands + 4,567.”

Korean (ko)
Uses Hanja numerals in traditional form: 一二三四五六七八九十百千萬億兆…, with readings like 만 (10,000), 억 (100,000,000), 조 (10¹²).

Large numbers again follow a 10⁴ structure: 60,000 = 六萬 (육만), 100,000 = 十萬 (십만), etc.

In a fully traditional notation, 1,234,567 would be written using Hanja blocks around 萬/億 units rather than 3‑digit commas.

Arabic (ar)
Traditional usage employs Eastern Arabic (Arabic‑Indic) digits: ٠١٢٣٤٥٦٧٨٩.

These digits are used in place of 0–9; grouping can be by thousands but with native punctuation or none in older manuscripts (structuring is often implicit in the text).

Indic languages (hi, bn, ta, etc.)
Each major Indic script has its own native numeral set, e.g. Devanagari ०१२३४५६७८९, Bengali ০১২৩৪৫৬৭৮৯, Tamil ௧௨௩ (plus distinct symbols for 10, 100, 1000 in older Tamil).

Traditional documents in those scripts would write numbers with these glyphs; Tamil historically did not use a pure positional 0–9 system but had special characters for 10/100/1000, etc.

Russian (ru) – historical note
Before adoption of “Arabic” digits, Russian used Cyrillic numerals, where letters carried numeric values (А=1, В=2, …) and special titlo marks denoted numbers.

This system is now obsolete outside of liturgical/archaic contexts, but it is the traditional non‑Latinized form.

If you want “traditional” formatting rules per non‑Western language, you would essentially:

Replace 0–9 with the language’s native numeral set (Chinese characters, kanji, Hanja, Eastern Arabic, Indic numerals).

For CJK, represent magnitudes explicitly with 万/億/兆 (or 万/억/조), i.e. think in 4‑digit blocks rather than thousand‑groups.
